ITSM / gestion de parc

Déploiement GLPI

Installation et configuration d'une plateforme GLPI sur serveur Linux afin de structurer le support, l'inventaire et la gestion d'incidents.

Stack
GLPI, Ubuntu Server, Apache, PHP, MariaDB
Objectif
Mettre en place une base ITSM et gestion de parc
Statut
Projet à compléter avec captures et rapport si disponibles

Contexte

GLPI est un outil pertinent pour formaliser le support, centraliser les tickets et structurer l'inventaire d'un parc informatique.

Objectifs

  • Installer la stack web sur Ubuntu Server.
  • Préparer Apache, PHP et MariaDB.
  • Configurer GLPI et ses prérequis.
  • Documenter les erreurs rencontrées et les corrections.

Workflow

  1. Installer les paquets système.
  2. Créer la base MariaDB et les droits associés.
  3. Déployer GLPI dans le répertoire web.
  4. Finaliser l'installation depuis l'interface.
  5. Vérifier l'accès, les plugins et les premiers objets.

Résultats

Le projet met en avant une compétence utile en entreprise : installer, diagnostiquer et documenter une solution de support et de gestion de parc.

Améliorations prévues

Ajouter la supervision, l'inventaire automatisé, des rôles utilisateurs et une documentation d'exploitation.
