ITSM / gestion de parc

Deploiement GLPI

Installation et configuration d'une plateforme GLPI sur serveur Linux afin de structurer le support, l'inventaire et la gestion d'incidents.

Stack
GLPI, Ubuntu Server, Apache, PHP, MariaDB
Objectif
Mettre en place une base ITSM et gestion de parc
Statut
Projet a completer avec captures et rapport si disponibles

Contexte

GLPI est un outil pertinent pour formaliser le support, centraliser les tickets et structurer l'inventaire d'un parc informatique.

Objectifs

  • Installer la stack web sur Ubuntu Server.
  • Preparer Apache, PHP et MariaDB.
  • Configurer GLPI et ses prerequis.
  • Documenter les erreurs rencontrees et les corrections.

Workflow

  1. Installer les paquets systeme.
  2. Creer la base MariaDB et les droits associes.
  3. Deployer GLPI dans le repertoire web.
  4. Finaliser l'installation depuis l'interface.
  5. Verifier l'acces, les plugins et les premiers objets.

Resultats

Le projet met en avant une competence utile en entreprise : installer, diagnostiquer et documenter une solution de support et de gestion de parc.

Ameliorations prevues

Ajouter la supervision, l'inventaire automatise, des roles utilisateurs et une documentation d'exploitation.
